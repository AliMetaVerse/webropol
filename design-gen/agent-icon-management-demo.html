Agent Icon Requirements

Three assets, one brand system, zero guesswork.

Microsoft 365 Copilot agents are packaged like Apps for Microsoft 365. That means one 192 x 192 color icon and two 32 x 32 PNG icons for default and focused states. This board keeps those assets separate so you can experiment with real uploads while staying inside the documented safe-area, contrast, and border rules.

Safe area first

Keep the core logo within the centered 120 x 120 safe area for the 192 icon. The extra perimeter gives Microsoft space to apply masking cleanly.

No rounded corners or borders

Upload a square PNG without a border. Corners and surface treatments are handled by Microsoft 365 automatically.

Design for reduction

The 32 px icons should survive extreme downscaling. Avoid long words, tiny detail, skewed marks, or low-contrast fills.

What this demo is modeling

  • The color icon uses a 192 x 192 square asset and shows a guide for the recommended safe region.
  • The default and focused placeholders stay separate so you can test a subtle rest state versus an emphasized active state.
  • The same uploaded assets propagate instantly to the mock Microsoft surfaces below the editor.
  • The export buttons save the whole board for review or handoff discussions.
Source reference: Microsoft Learn article “Design icons for agent acquisition and management,” last updated 2025-11-20.

Suggested workflow

  • Drop in your final square brand icon first and check the large safe-area preview.
  • Upload simplified 32 px defaults if the color icon loses clarity when scaled down.
  • Create a slightly stronger focused icon rather than inventing a different symbol.
  • Export the board once the store tile and app bar previews feel balanced together.
