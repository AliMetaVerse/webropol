Context Surfaces

Tooltips for hints. Popovers for action.

These components share one visual system but solve different jobs. Use tooltips for quick explanatory copy on hover or focus. Use popovers when the content needs buttons, choices, or a short workflow.

Do and Don't

The difference is behavioral, not decorative. Choose the lighter surface only when the interaction model stays light.

Do

Use the right surface for the job

Use tooltips for one concise sentence that explains an icon, setting, or control.
Use popovers when content needs buttons, multiple choices, links, or a short workflow.
Keep both surfaces anchored to a clear trigger and place them near the triggering element.
Let long popovers scroll internally on smaller screens instead of allowing content to clip.
Don't

Don't overload the pattern

Don't put interactive controls inside a tooltip. If it needs a click target, it should be a popover.
Don't use popovers for critical confirmation or large forms. Escalate to a modal when the task grows.
Don't rely on container overflow being visible. These components should float independently above layout regions.
Don't write tooltip copy as paragraphs. Keep it short enough to scan instantly.

API Notes

Both components use the same default slot for the trigger element. Popover additionally accepts nodes marked with `slot="panel"` or `data-popover-panel` for rich panel content.

Tooltip

Short help surface

Best for a sentence of guidance with no interactive controls.

textTooltip body copy. `content` is supported as an alias.
position`top`, `bottom`, `left`, `right`, plus `top-start`, `top-end`, `bottom-start`, `bottom-end`.
variant`default`, `subtle`, or `brand` using the same neutral-teal system as popover.
trigger`hover` by default, with `click` and `both` also supported.
<webropol-tooltip text="Explains the action" position="top" variant="brand"> <button>Hover me</button> </webropol-tooltip>
Popover

Interactive contextual panel

Use when the surface needs actions, lists, or a close affordance.

titleOptional header title rendered above the panel content.
textOptional supporting copy below the title.
positionSame placement model as tooltip. Default is `bottom-start`.
variant`default`, `subtle`, or `brand` for the shared visual system.
closableAdds a close button in the header.
openBoolean attribute for controlled/manual state.
scrollingPopover body only becomes scrollable when the content exceeds the available viewport space.
<webropol-popover title="Share survey" position="bottom-start" closable> <button>Open menu</button> <div slot="panel">...rich content...</div> </webropol-popover>
